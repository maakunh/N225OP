# N225OP — 日経225ミニオプション IV監視ダッシュボード

楽天証券 **MarketSpeed II RSS** で取得した日経225ミニオプションのデータを Excel に時系列で蓄積し、ブラウザ上の4つのダッシュボード（IVスマイル・価格/IVマルチチャート・IVヒートマップ・先物/NT倍率）でリアルタイムに可視化するツールです。

権利行使価格ごとの **現在値** と **IV（インプライド・ボラティリティ）** を1分ごとに自動記録し、市場が織り込む「期待のブレ幅」とその時間変化を一目で把握することを目的としています。

> **ライブラリ非依存の設計**：データ取得は Excel + VBA、可視化は素の HTML + JavaScript（Chart.js のみ CDN 利用）。特別な開発環境やビルド工程は不要です。

---

## 主な機能

- **自動データ取得**：MarketSpeed II RSS の `RssFOPCode` / `RssFOPMarket` 関数で全権利行使価格のコール／プット現在値・IVを取得
- **時系列ログ化**：VBA が1分ごとにスナップショットを「Log」シートへ縦持ちで蓄積
- **自動エクスポート**：数分ごとにブラウザが読める `data.js`（1分足）と `data_hourly.js`（1時間足・最大30日）を自動書き出し（OneDrive 対応の固定フォルダ方式）
- **4つのダッシュボード**
  - **IVスマイル** — 権利行使価格×IVの曲線。時刻スライダー／再生で時間変化を確認。OTM側IV採用・下方スキュー表示。1時間足・最大30日表示（v1.1.3）
  - **価格/IVマルチチャート** — 行使価格ごとの推移を小チャートで格子状に一覧。ATM中心に表示本数可変。**画面上部のトグルで「価格」⇔「IV」をワンクリック切替**（v1.1）し、コール／プット両系列の表示単位・凡例・最終値ラベルが一括で連動。**ATM は先物ミニ現在値から自動判定し、ATMバッジで明示**（v1.1）。1分足表示
  - **IVヒートマップ** — 行使価格×時刻のIVをカラースケールで表現。コール／プット／OTM側を切替。1時間足・最大30日表示、ATM は先物ミニ現在値から自動判定（v1.1.3）
  - **先物/NT倍率** — 日経225ミニ先物（5分足/24時間）、日経225先物・TOPIX先物（各 日足/1年・60分足/30日）をローソク足で、NT倍率（1年）を曲線で表示（v1.2）。`RssChart` で取得し `ExportFutures` で手動更新
- **自動更新**：各ダッシュボードはオプションで自動リロードに対応
- **ストール検知**：G1（取得時刻）の更新停止を監視し、データが止まると各画面上部にアラートバナーを表示
- **実行ログ**：主要マクロ（開始/停止/先物更新/ダッシュボード表示）の実行を `RunLog` シートに「日時・機能・備考」で記録。最大行数に達すると古いものから自動削除

```

        ▼                         ▼

        ▼                         ▼

        （ブラウザで表示）

        │  ExportFutures を手動実行

        │
        ▼
先物/NT倍率ダッシュボード（ブラウザで表示）
```

## セットアップ

### 必要環境

- 楽天証券の口座と **MarketSpeed II**（RSS機能を含む）
- **Microsoft Excel**（デスクトップ版、Microsoft 365 / 2019 以降推奨）
- **Webブラウザ**（Microsoft Edge または Google Chrome）
- インターネット接続（Chart.js を CDN から読み込むため）

### 手順（概要）

1. **MarketSpeed II RSS を有効化**し、Excel で RSS 関数が使える状態にする
2. Excel ブックを作成し、**マクロ有効ブック（.xlsm）** として保存
3. シートを2枚用意する
   - **Live**：RSS関数を並べてリアルタイム表示（A=権利行使価格, B=Cコード, C=C現在値, D=C_IV, E=Pコード, F=P現在値, G=P_IV、データは4行目から）
   - **Log**：時系列蓄積用（1行目ヘッダー：時刻／権利行使価格／C現在値／C_IV／P現在値／P_IV）
4. `excel/OptionLogger.bas` を VBエディタ（`Alt + F11`）の「ファイル」→「ファイルのインポート」で取り込む
5. `OptionLogger.bas` の定数 `DASH_FOLDER`（既定 `C:\OptionDash`）を出力先に設定
6. `web/` の3つのHTMLを `DASH_FOLDER` に配置（`data.js`・`data_hourly.js` は自動生成される）

> ダッシュボードは `data.js` と同じフォルダに置かれている必要があります。リポジトリはソース管理用にフォルダ分けしていますが、実際の運用では3つのHTMLを `DASH_FOLDER`（既定 `C:\OptionDash`）に集め、そこに VBA が `data.js` を書き出します。

詳細は [`docs/implementation_guide.md`](docs/implementation_guide.md) を参照してください。

### 銘柄コードの生成（RSS関数）

日経225ミニオプションの銘柄コードは `RssFOPCode` 関数で自動生成します。

```
=RssFOPCode("N225MOP", "202507", 1, A4)   ' コール（C/P区分=1）
=RssFOPCode("N225MOP", "202507", 2, A4)   ' プット（C/P区分=2）
```

- 第1引数 **銘柄種類**：日経225ミニオプションは `N225MOP`
- 第2引数 **限月**：`202507` 形式（ウィークリーは `202507#2` のように `#n` を付与）
- 第3引数 **C/P区分**：コール=`1`、プット=`2`
- 第4引数 **行使価格**：セル参照

生成した銘柄コードから現在値・IVを取得します。

```
=RssFOPMarket(B4, "現在値")
=RssFOPMarket(B4, "IV")
```

## 使い方

| マクロ | 動作 |
|---|---|
| `StartLogging` | ログ取得を開始（1分ごとに記録、数分ごとに `data.js` を更新） |
| `OpenDashboards` | `data.js` を最新化してブラウザで IVスマイルを開く |
| `StopLogging` | ログ取得を停止 |
| `ForceStopLogging` | 緊急停止（スケジュールを総当たりで解除） |
| `ExportFutures` | 先物/NT倍率データを `RssChart` で取り直し `data_futures.js` を書き出す（手動・別系統） |
| `StartFutures` | 先物/NT倍率の自動更新を開始（60分ごとに `ExportFutures` を実行） |
| `StopFutures` | 先物/NT倍率の自動更新を停止 |

1. MarketSpeed II を起動・ログイン
2. Excel ブックを開き、Live シートに値が出ていることを確認
3. `StartLogging` を実行（先物も自動更新するなら続けて `StartFutures` も実行）
4. `OpenDashboards` を実行 → ブラウザでダッシュボードが開く
5. 画面上部のナビで4つのダッシュボードと使い方ガイドを行き来できる
6. 各画面右上の「自動更新」で5分ごとの自動リロードを切替

### 先物/NT倍率ダッシュボードの準備

先物画面はオプション系とは独立した別系統です。`Chart` という名前のシートを用意し、2行目の各列に `RssChart` 式を配置します（銘柄コードは Live シート参照）。

銘柄コードを置く Live シートのセルは次の通りです。

| セル | 内容 |
|---|---|
| H1 | 日経225ミニ先物の銘柄コード |
| I1 | 日経225先物（ラージ）の銘柄コード |
| J1 | TOPIX先物の銘柄コード |
| C1 | メジャーSQ限月（3・6・9・12月） |

> **ラージ・TOPIX先物はメジャーSQ限月でないと銘柄コードを取得できません。** そのため通常限月（B1）とは別に、メジャーSQ限月を **C1** に持たせ、**I1・J1 は C1 を参照**して銘柄コードを生成しています（ミニ先物の H1 は通常限月ベース）。限月をまたぐ際は C1 のメジャーSQ限月を更新してください。

| セル | 式 | 系列 |
|---|---|---|
| A2 | `=RssChart(,Live!H1,"5M",300)` | ミニ先物 5分足 |
| L2 | `=RssChart(,Live!I1,"D",260)` | ラージ 日足 |
| W2 | `=RssChart(,Live!I1,"60M",500)` | ラージ 60分足 |
| AH2 | `=RssChart(,Live!J1,"D",260)` | TOPIX 日足 |
| AS2 | `=RssChart(,Live!J1,"60M",500)` | TOPIX 60分足 |

各系列は11列間隔で配置（10列展開＋余白1）。データが展開されたら `ExportFutures` を実行すると `data_futures.js` が書き出され、`futures_dashboard.html` で表示できます。NT倍率はラージ日足÷TOPIX日足で自動算出されます。

`ExportFutures` は手動実行ですが、`StartFutures` を実行すると60分ごとに自動更新されます（停止は `StopFutures`）。1分ごとのオプションログ（`StartLogging`）とは独立した別タイマーで動きます。

> 列やシート名を変えたい場合は `OptionLogger.bas` 冒頭の `FUT_SHEET` / `COL_*` 定数で調整できます。RssChart のデータ展開開始行が異なる場合は `FUT_FORMULA_ROW` を合わせてください。

## ダッシュボードの見方

オプション初心者向けの使い方ガイドを同梱しています（[`docs/guide.md`](docs/guide.md)）。IVとは何か、スキューの読み方、3画面の組み合わせ方、シグナルの売買への結びつけ方までを解説しています。要点は次のとおりです。

- **IVスマイル**：曲線全体が上がればIV上昇（割高方向）、下方ほどIVが高い右肩下がりは下落ヘッジ需要の表れ（スキュー）
- **価格/IVマルチ**：トグルが「価格」のとき、青（コール）が一斉上昇＝上昇局面、桃（プット）が一斉上昇＝下落局面。「IV」に切り替えると同じ格子で各行使価格のIV推移を比較でき、青・桃がそろって上昇＝IV全面上昇（荒れ）と読める。線の途切れは流動性が低い行使価格
- **ヒートマップ**：横方向に赤くなる＝時間とともにIV上昇、特定の列だけ赤＝その時刻のイベント

## 設定値

`OptionLogger.bas` 冒頭の定数で調整できます。

| 定数 | 既定値 | 意味 |
|---|---|---|
| `INTERVAL` | `"00:01:00"` | ログ取得間隔 |
| `FIRST_DATA_ROW` | `4` | Liveシートのデータ開始行 |
| `EXPORT_EVERY` | `5` | ログ何回ごとに `data.js` を更新するか |
| `DASH_FOLDER` | `"C:\OptionDash"` | ダッシュボード一式の出力先（OneDrive外推奨） |
| `FUT_INTERVAL` | `"01:00:00"` | 先物自動更新の間隔（`StartFutures`） |
| `RUNLOG_MAX` | `10000` | 実行ログ（RunLogシート）の最大行数。超えたら古い行から削除 |

## 技術メモ

- **OneDrive対応**：ブックが OneDrive 配下だと `ThisWorkbook.Path` が URL を返し書き出しに失敗するため、`data.js` は OneDrive 外の固定フォルダへ出力する設計
- **文字コード**：`data.js` は `ADODB.Stream` で UTF-8 出力（文字化け防止）
- **書き込みリトライ**：同期ロック対策として `data.js` 書き出しは最大3回リトライ
- **「0」の扱い**：板なし（取引・気配なし）の現在値0は、グラフ上で線を繋がず欠損として描画
- **CDN依存**：Chart.js を `cdn.jsdelivr.net` から読み込むためオンライン環境が前提（ヒートマップのみ CDN 不要で自己完結）
- **ストール検知**：VBA が G1 の最終更新実時刻を `data.js` の `OPTION_META.lastAlive` に書き出し、各ダッシュボードがブラウザのリアルタイム時計と比較して停止を判定（しきい値は各 HTML の `STALL_SEC`、既定420秒）。自動リロード間隔より長く設定する必要がある
- **ATM動的判定**：VBA が先物ミニ現在値（Live!F1）を `OPTION_META.underlying` に書き出し、価格/IVマルチチャートと IVヒートマップが最近接の行使価格を ATM として算出。取得不可（空・エラー・0）時は既定値にフォールバック
- **1時間足の生成**：`ExportDataJsHourly` が `Log` を「日付＋時＋行使価格」ごとの最終スナップショットに畳み込み、正時表記で `data_hourly.js`（`OPTION_DATA_H`）へ出力。`Scripting.Dictionary` の後勝ち上書きで各時間の最終値を採用しつつ初出順（時系列順）を保持。直近30日より古い行は除外
- **先物データの生成**：`ExportFutures` が `Chart` シートの `RssChart` 展開結果（4本値）を読み、OHLCを `data_futures.js`（`FUTURES_DATA`）へ出力。NT倍率はラージ日足とTOPIX日足の終値を日付キーで突き合わせて比を算出。オプション系（data.js / data_hourly.js）とは独立した別系統で、手動実行・毎回取り直し
- **実行ログ**：`WriteRunLog` が主要マクロの実行を `RunLog` シートに追記。データ行数が `RUNLOG_MAX` を超えるとヘッダー直下（最古）から超過分を削除してローテーション。記録失敗が本体処理を止めないよう `On Error Resume Next` で保護。毎分の `LogSnapshot` や `ExportDataJs` は記録対象外（ログ肥大を避けるため）

### v1.1
- **価格/IVマルチチャートにトグル切替を追加** — 従来の価格専用「価格マルチチャート」を「価格/IVマルチチャート」に拡張。画面上部の「価格／IV」トグルで、格子状の全セルの表示を価格⇔IVへワンクリックで一括切替できるようにした

- **ATM の動的判定と ATM バッジを価格/IVマルチチャートに追加** — 先物ミニ現在値（Live シートの F1）を原資産として、最も近い権利行使価格を ATM として自動判定するようにした（従来は固定値 70000）
  - VBA が F1 の値を `data.js` の `OPTION_META.underlying` に書き出し、ブラウザが最近接の行使価格を ATM として算出。原資産が動けば ATM ハイライト・ATM バッジ・表示銘柄の中心（ATM 基準）がすべて追従する
  - ATM セルの見出しに「ATM」バッジを表示。ステータス欄に現在の ATM 行使価格と先物ミニ価格を併記
  - F1 が空・エラー・0 の場合は既定値 70000 にフォールバック
- これに伴いドキュメント（README・guide）の名称と解説を「価格/IVマルチチャート」基準に更新
